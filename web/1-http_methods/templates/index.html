Where are the other parts?

There are 9 unique HTTP methods, but I have chosen the 6 most frequently used to hide the flags. You get a part when you make a successful HTTP request with that method. If you want to know all 9 you can check the summaries Here


  1. GET This method is what your browser sends to get the contennt of a web page or the requested resource.
  2. POST This method is used to send data to a specific destination. An example of this is a login form uses a POST method to submit your username and password to the server. When you send this request you should include a body after the request headers.
  3. OPTIONS This method is used to find out which HTTP methods a resource expects.
  4. HEAD This method returns only the headers from a GET request. This is a fastway for checking status codes before continuing a long part of a script.
  5. PUT This method writes data to the requested resource. This is useful to write your own exploit code.
  6. DELETE This method deletes the specified resource.

How do you make other types of requests

The easiest way to do this is with a graphical program like Postman. But if you want to be a 1337 h4x0r you can use a command line tool called curl like so: curl -X [HTTP METHOD] www.google.com or the python requests library or pwntools

All methods that can have a body check that the bobdy is not null so make sure you send any data. If you're using curl look into the -d flag.
